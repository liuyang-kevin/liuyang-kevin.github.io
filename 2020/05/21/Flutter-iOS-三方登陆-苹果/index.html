苹果登陆 Sign in with Apple

按照使用途径可分为，

  • iOS手机登陆
    • 手机iOS客户端登陆开发者后台配置流程下面提到
    • https://www.jianshu.com/p/483b998f2370
  • web登陆
    • 通过网站restful登陆，Android，pc端等使用，开发者后台需要额外配置
  • 介绍 iOS / web的
    后端链接苹果服务器，又分为两种方式
  • 根据iOS客户端返回的 identityToken user 传给后端，后端去验证
  • 根据开发者后台配置，生成的私钥文件key（一次性生成）与苹果后端验证
  • 介绍后端接入的

    代码实现

    一. 开启Sign in with Apple 功能

  1. app bundle id 开启Sign in with Apple，登陆developer账号，在app bundle ID的Capabilities里，打勾Sign In with Apple.
  2. Xcode里开启Sign in with Apple， 项目设置 -> Signing & Capabilities 里，开启Sign in with Apple选项。
  3. 实现分四大部分：
    • 创建Sign in with Apple Button.
    • 跟用户提出授权请求.
    • 根据用户的授权来验证用户.
    • 处理用户授权变更.

  • handleAuthorization里主要是跟用户提出用苹果登陆请求，并要求用户提供用户名和email.
  • 发出请求需要创建ASAuthorizationController, 但是需要提供delegate 和 presentationContextProvider.

    其他注意点。

    使用者角度

  1. iOS，Mac，有appleId；
  2. 消账号授权
    1. 点击手机中的「设置」，点击你的「Apple账户」，然后接着点击进入「密码与安全性」。
    2. 将会看到一个选项，叫「使用您的 Apple ID 的 App」，点击进入，将会看到已经授权登录的APP应用列表们。
    3. 选择你想要取消授权的APP应用，如点击这个应用，会进入到详情页面，可以看到有「停止使用 Apple ID」的按钮，点击即可取消授权。

      开发者角度 / 产品接入「Sign in with Apple」

      ToC端产品，基本上就是这句话概括了

      对于新上架的应用，支持第三方登录方式的都基本要接入，不接入的话是会苹果所被拒绝的。

符合以下条件的 App，可以不接入：

  1. 使用自建账户和登录系统；
  2. 要求用户使用现有的教育或企业账户登录的教育、企业或商业类应用；
  3. 使用政府或行业支持的公民身份识别系统或电子 ID 来验证用户；
  4. 应用特定于第三方服务，用户需要使用邮箱、社交媒体或其它第三方账户才能访问其内容的应用。

需要有「入口」——「授权请求」——「验证」——「验证结果」——「处理变更」五个流程步骤。

授权的操作过程有两步：

  1. Apple ID授权；
  2. 苹果的双重验证。

回调数据分为：

  • User ID——苹果用户唯一标识
  • 验证数据： Identity token, code验证数据
  • 苹果用户基础信息返回
  • Real user indicator用于判断当前登录的苹果账号是否是一个真实用户，取值有：unsupported、unknown、likelyReal

按钮规范

一句话：图标显眼，放在头一个，用官方给的图

Apple图标传送门

标识使用问题：

  • 切勿单独使用Apple图标作为按钮。
  • 使Apple图标文件的高度与按钮的高度匹配。
  • 不要裁剪图标文件。
  • 不要添加渐变填充。
  • 不要在图标文件中使用自定义颜色。
